本系统为真实机器人AI控制系统：
控制链采用 policy_runtime_service + robot_client 的闭环执行架构。
mock_task 仅用于UI兜底显示，不参与任何机器人控制。

一、前端在系统中的位置

前端是 SO-ARM101 苹果采摘系统的 UI 层，只负责展示状态和发送任务意图。
前端不能直接控制机器人，不能生成机械臂 action，也不能绕过后端调用硬件。

真实控制链必须是：

frontend
  -> FastAPI backend
  -> task_control
  -> policy_runtime_service
  -> LeRobot ACT policy inference
  -> robot_client
  -> SO-ARM101 robot execution

二、UI 展示链

前端接收的是展示链数据：

robot_status + backend_state + optional mock fallback
  -> status_fusion
  -> websocket / HTTP
  -> frontend UI

mock fallback 只用于 UI 兜底显示，不参与 action/control。

三、local / remote 的统一含义

local 模式：

backend -> policy_runtime_service（本机进程） -> robot_client -> SO-ARM101

remote 模式：

backend -> HTTP -> Robot PC -> policy_runtime_service（远程进程） -> robot_client -> SO-ARM101

local 和 remote 都进入 policy_runtime_service + robot_client 闭环，二者只是同一套真实 AI 控制系统的不同部署方式。

四、前端需要调用的接口

- POST /set_target_apple：设置目标采摘模式。
- POST /start_task：启动任务，请求体携带 mode。
- POST /stop：停止任务，请求体携带 mode。
- POST /reset：复位后端任务状态。
- GET /status：获取融合后的状态。
- GET /logs：获取运行日志。
- WebSocket：接收后端推送的融合状态。

五、前端展示数据约束

电量、天气、果筐容量、剩余时间、区域、路线、异常提示等页面数据必须来自后端状态源或明确标注为 UI fallback。
如果使用 mock fallback，只能用于页面显示，不能被解释为控制系统已经执行真实动作。

六、运行方式

第一次运行：

npm install
npm run dev

默认地址：

http://localhost:5173/

七、主要文件

src/App.jsx：主 UI 和交互逻辑。
src/api.js：后端 API 封装。
public/：Logo 和图标资源。
package.json：依赖和脚本。
vite.config.js：Vite 配置。
